Quiero implementar el modelo completo de monetización y roles de ServiLink basado en un sistema escalable con clientes, trabajadores, vendedores y hosts.

Este cambio es estructural (backend + frontend + lógica de negocio), así que hazlo de forma ordenada y sin romper lo existente.

🔹 1. ROLES DEL SISTEMA

Crear y usar estos roles claramente separados:

customer → cliente que contrata servicios o compra productos
worker → trabajador que presta servicios
seller → vendedor del ServiMarket (dueño de tienda)
host → gestor que gana comisiones por operaciones

IMPORTANTE:

Un usuario puede tener más de un rol (multi-role), pero cada rol tiene lógica independiente
NO mezclar seller con host
🔹 2. MODELO DE COMISIONES
🛠 SERVICIOS

Cuando se paga un servicio:

Comisión total: 20%

Distribución:

12% → plataforma
8% → host (si existe)
80% → trabajador

Si NO hay host:

20% → plataforma
80% → trabajador
🏪 PRODUCTOS (ServiMarket)

Cuando se vende un producto:

Comisión total: 10%

Distribución:

7% → plataforma
3% → host (si gestiona esa tienda)
90% → vendedor (seller)

Si NO hay host:

10% → plataforma
90% → vendedor

IMPORTANTE:

El seller NUNCA gana comisión (solo vende)
🔹 3. RELACIÓN HOST

Implementar relación opcional:

Un worker puede tener hostId
Una store puede tener hostId

Esto define si el host gana comisión o no.
