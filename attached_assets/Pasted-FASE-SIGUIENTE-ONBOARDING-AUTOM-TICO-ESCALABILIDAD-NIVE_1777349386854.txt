FASE SIGUIENTE — ONBOARDING AUTOMÁTICO + ESCALABILIDAD (NIVEL PLATAFORMA)

OBJETIVO:
Convertir el Sync Agent + LinkServi en un sistema que:

* se active casi sin intervención humana
* escale a múltiples negocios sin soporte manual
* sea resiliente y observable en producción

RESTRICCIONES:

* NO modificar la lógica de sincronización existente
* NO cambiar /api/integrations/products/sync
* Construir encima: onboarding, orquestación, validación y observabilidad

---

T001 — ONBOARDING REMOTO AUTOMÁTICO (PAIRING)

Implementar flujo “pairing” desde LinkServi:

1. En LinkServi (web):

* botón: “Conectar mi sistema”
* genera pairingCode (UUID corto, TTL 10 min)
* muestra código QR + código alfanumérico

2. En Sync Agent (primer arranque):

* pantalla: “Conectar con LinkServi”
* opción: escanear QR (webcam) o pegar código

3. Endpoint backend:
   POST /api/integrations/agent/pair
   Body: { pairingCode }

* valida código
* devuelve apiKey + apiUrl + storeId

4. Guardar automáticamente en config.json
5. Marcar pairingCode como usado (one-time)

RESULTADO:
El usuario no copia API keys. Conecta en 1 paso.

---

T002 — AUTO-DETECCIÓN AVANZADA DE DB (SIN CAMPOS)

Implementar motor de detección:

* escanear localhost:1433 y puertos comunes
* listar instancias SQL Server disponibles
* intentar login con credenciales Windows (si aplica)
* descubrir bases con tablas tipo “productos”

Endpoint local:
GET /api/db/autodetect-advanced
Respuesta:
{
candidates: [
{ host, port, database, confidenceScore, tablesDetected }
]
}

UI:

* seleccionar automáticamente el mejor candidato (confidenceScore)
* fallback manual si ninguno válido
